# Roadmap

Buselligence v8 establishes the kernel primitive. Future work focuses on ecosystem growth, not feature sprawl.

## Shipped (v8)

- [x] Buselligence Kernel — unified execution layer
- [x] Skills system
- [x] Agent registry and lifecycle
- [x] Evaluation framework
- [x] AI observability (traces)
- [x] Cost intelligence (BYOK)
- [x] buselligence.lock reproducible environments
- [x] Extension SDK
- [x] CLI (`bus create`, `deploy`, `evaluate`)
- [x] Hello World experience (`/start`)
- [x] Reference examples and documentation
- [x] Local-first architecture

## Near term

- [ ] Publish `@buselligence/cli` to npm
- [ ] Publish `@buselligence/sdk` package
- [ ] Community hub UI (agent/skill marketplace)
- [ ] PostgreSQL adapter for production scale
- [ ] Docker Compose one-liner
- [ ] More MCP marketplace presets

## Medium term

- [ ] Desktop app (Electron/Tauri)
- [ ] Skill publishing workflow
- [ ] Agent versioning and rollback
- [ ] Multi-tenant deployments
- [ ] Hosted cloud option (optional)

## Long term

- [ ] Buselligence Cloud (managed runtime)
- [ ] Skill registry on npm
- [ ] Certification program for community agents
- [ ] Enterprise governance pack

## Principles (unchanged)

1. **One primitive** — the kernel, not 200 features
2. **Open source first** — MIT, community-owned
3. **Local-first** — your AI runs anywhere
4. **Developer experience** — 60 seconds to wow
